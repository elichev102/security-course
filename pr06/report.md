# ПР №6. Анализ сетевого трафика: tcpdump и Wireshark

## 1. Разбор строки tcpdump

**Строка:** `12:34:56.789012 IP 127.0.0.1.44321 > 127.0.0.1.8080: Flags [S], seq 123, length 0`

| Поле | Значение | Что означает |
|------|----------|--------------|
| 12:34:56.789012 | Время | Точное время захвата пакета |
| IP | Протокол | Используется протокол IP |
| 127.0.0.1.44321 | Источник | IP-адрес и порт отправителя |
| 127.0.0.1.8080 | Назначение | IP-адрес и порт получателя |
| Flags [S] | Флаг SYN | Начало установки TCP-соединения |
| seq 123 | Последовательность | Начальный номер последовательности |
| length 0 | Длина | Длина данных (0 байт) |

---

## 2. Перехваченные данные HTTP

**Что видно в трафике:** Пароль `supersecret123` и логин `admin` передаются в открытом виде в теле HTTP-запроса.

**Кто в реальной сети может так же прочитать этот трафик:**
- Любой, кто находится в той же локальной сети (Wi-Fi, Ethernet)
- Сетевой администратор
- Злоумышленник, подключившийся к сетевому коммутатору
- Оператор интернет-провайдера

---

## 3. Follow TCP Stream

**Что видит злоумышленник перехвативший этот трафик:**
Полный диалог клиент-сервер:
- Запрос клиента: POST /login с параметрами `username=admin` и `password=supersecret123`
- Ответ сервера: HTTP/1.1 501 Unsupported method

Скомпрометированы: логин, пароль, данные сессии.

---

## 4. HTTP vs HTTPS — сравнение

| Что наблюдаем | HTTP (порт 8080) | HTTPS (порт 443) |
|---------------|------------------|------------------|
| IP-адреса сторон | Видны | Видны |
| Порт назначения | Виден (8080) | Виден (443) |
| Заголовки HTTP | Видны | Не видны (зашифрованы) |
| Содержимое запроса (body) | Видно открытым текстом | Не видно (зашифровано) |
| Логин и пароль | Видны | Не видны |
| Имя домена (SNI) | Видно в Host: | Видно (SNI) |

---

## 5. DNS и приватность

**Какие домены видны в DNS даже при HTTPS:** 
- google.com
- github.com
- example.com

**Что это означает для приватности:** Даже если используется HTTPS, DNS-запросы показывают, какие сайты посещает пользователь. IP-адреса и размеры пакетов также утекают.

## 6. Base64

**Декодированная строка:** `dmFzeWE6TXlQYXNzd29yZDIwMjQ=` → `vasya:MyPassword2024`

**Почему Base64 не является шифрованием:** Base64 — это кодировка, а не шифрование. Она преобразует данные в текстовый формат, но не требует ключа для обратного преобразования. Любой может декодировать Base64 и прочитать исходные данные.

---

## 7. tshark vs tcpdump vs Wireshark

| Инструмент | Когда использовать |
|------------|-------------------|
| tcpdump | Быстрый захват в консоли, на сервере без GUI |
| tshark | Автоматизация, скрипты, анализ pcap-файлов без GUI |
| Wireshark | Глубокий анализ, визуализация, работа с протоколами |

---

## 8. Каналы утечки через сеть

**Какие меры защиты закрывают сетевой канал утечки:**
1. **HTTPS/TLS** — шифрование данных при передаче
2. **VPN** — шифрование всего трафика
3. **DNS-over-HTTPS (DoH)** — защита DNS-запросов
4. **Брандмауэр** — ограничение доступа к сети
5. **Обнаружение аномалий** — мониторинг необычного трафика

В ходе работы были изучены:

2. **Wireshark** — визуальный анализ пакетов

4. **Утечка DNS** — домены видны даже при HTTPS

Самым важным выводом стало то, что HTTP абсолютно небезопасен — пароли и данные передаются открытым текстом. HTTPS шифрует содержимое, но даже он не скрывает домены (DNS) и IP-адреса.
