### Поля /etc/passwd (разбор строки alice)

Строка из `getent passwd alice`:

Разбор по полям:
1. `alice` — имя пользователя
2. `x` — пароль зашифрован в /etc/shadow (признак теневого хранения)
3. `1001` — UID (идентификатор пользователя)
4. `1001` — GID (идентификатор основной группы)
5. (пусто) — комментарий (GECOS), обычно ФИО
6. `/home/alice` — домашняя директория
7. `/bin/bash` — командная оболочка

### /etc/shadow

В этом файле хранятся хэши паролей и политики их срока действия. Он недоступен обычным пользователям (права 640), потому что содержит чувствительные данные — если их украсть, злоумышленник может подобрать пароль перебором.

## 2. Права доступа chmod/chown

Итоговые права `/srv/project/code`: `drwxrwx---` (770), владелец `alice:developers`

- **bob** (в группе developers): при правах 750 **не мог** создать файл, при правах 770 **смог** создать файл `hello.py`
- **carol** (не в группе): **не смогла** войти в директорию при правах 770

### Права файла q1.txt
- Символьная запись: `-rw-r-----`
- Числовая запись: `640`
- Владелец `alice` — чтение и запись (rw- = 6)
- Группа `auditors` — только чтение (r-- = 4)
- Остальные — нет доступа (--- = 0)
## 3. ACL

Задача: дать carol доступ на чтение к папке `code` без изменения группы.

Команда: `setfacl -m u:carol:r-x /srv/project/code`

Вывод `getfacl` после изменения:

После добавления ACL в конце прав в `ls -la` появился символ `+`.

ACL лучше стандартной модели, потому что позволяет дать доступ конкретному пользователю (carol), не изменяя группу файлов и не давая доступ всей группе developers.
## 4. sudo-политики

| Пользователь | Разрешено | Ограничение |
|---|---|---|
| alice | Всё (NOPASSWD) | Нет |
| bob | apt, apt-get | Только пакетный менеджер |
| carol | journalctl, cat /var/log/* | Только просмотр логов |

**Выводы:**
- Alice использует `NOPASSWD` потому что она администратор — это ускоряет её работу.
- Bob и Carol используют `NOPASSWD` только для ограниченного набора команд.
- Реализован принцип **наименьших привилегий** — каждый получает только необходимые права.
