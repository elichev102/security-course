## 2. auditd — настройка правил

### Применённые правила (вывод auditctl -l)

### Постоянные правила из файла security-course.rules

В файл `/etc/audit/rules.d/security-course.rules` добавлены правила для:
- Мониторинга файлов аутентификации (`/etc/passwd`, `/etc/shadow`, `/etc/group`)
- Мониторинга привилегий (`/etc/sudoers`, `/etc/sudoers.d/`)
- Мониторинга SSH конфига (`/etc/ssh/sshd_config`)
- Фиксации отказов в доступе
- Фиксации удаления файлов
- Фиксации повышения привилегий

### Разбор записи SYSCALL

| Поле | Значение | Что означает |
|------|----------|--------------|
| auid | 1000 | Аутентификационный UID (оригинальный пользователь, залогинившийся в систему) |
| uid | 0 | Эффективный UID процесса (root) |
| euid | 0 | Сохраняемый UID (тоже root) |
| comm | "cat" | Имя команды, вызвавшей системный вызов |
| exe | "/usr/bin/cat" | Полный путь к исполняемому файлу |

## 3. Расследование инцидента

### Хронология действий bob

| Время | Действие | Результат | Источник (команда поиска) |
|-------|----------|-----------|---------------------------|
| ~18:40 | cat /etc/shadow | Отказ | ausearch -k access-denied |
| ~18:40 | ls /root | Отказ | ausearch -k access-denied |
| ~18:40 | cat /etc/passwd | Успех (разрешено) | ausearch -ua bob |
| ~18:40 | touch /tmp/totally-not-malware.sh | Успех | ausearch --start recent |
| ~18:40 | cat /etc/sudoers | Отказ | ausearch -k access-denied |

### Как auditd помог расследованию

Auditd позволил:
1. Увидеть все попытки доступа к защищённым файлам через ключ `access-denied`
2. Отследить конкретные действия пользователя bob по его UID
3. Найти факт создания подозрительного файла
4. Зафиксировать изменение sudoers через ключ `priv-files`

### Чем auditd лучше journalctl для задач аудита безопасности

| Критерий | journalctl | auditd |
|----------|------------|--------|
| Уровень | Пользовательский (логи приложений) | Ядерный (системные вызовы) |
| Детализация | Только что написало приложение | Каждый системный вызов |
| Обход | Приложение может скрыть логи | Приложение не может скрыть syscall |
| Правила | Нет, только фильтрация | Есть, можно настраивать |

## Выводы

В ходе работы были изучены:
1. **journalctl** — просмотр системных журналов, фильтрация по времени, сервисам и уровню ошибок
2. **auditd** — настройка правил аудита на уровне ядра, мониторинг файлов и системных вызовов
3. **Расследование инцидента** — поиск следов действий пользователя через ausearch
4. **Связь с нормативкой** — реализованы меры аудита согласно Приказу ФСТЭК №17

Auditd является более мощным средством для ИБ, чем journalctl, так как работает на уровне ядра и не может быть обойдён приложением. Это критично для расследования инцидентов и выполнения требований регуляторов.
